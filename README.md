# Web Pikaso

سایت خدماتی طراحی و توسعه وب — ساخته‌شده با Next.js (App Router)، React، Node.js (API Routes)، TypeScript و Tailwind CSS. دو زبانه (فارسی/انگلیسی، پیش‌فرض فارسی و RTL) و دو حالت نمایش (تاریک/روشن، پیش‌فرض تاریک).

## اجرای پروژه

```bash
npm install
cp .env.example .env.local   # و مقدار JWT_SECRET را عوض کنید
npm run dev
```

سپس آدرس `http://localhost:3000` را باز کنید.

برای build نهایی:

```bash
npm run build
npm run start
```

> این پروژه در محیطی بدون دسترسی به اینترنت ساخته شده، پس امکان اجرای `npm install` و تست build در همین‌جا نبود. کد کامل و آماده اجراست؛ فقط لازم است روی سیستم خودتان `npm install` را اجرا کنید.

## ساختار پروژه

```
app/
  page.tsx                  صفحه اصلی (Hero, اعتماد, نمونه‌کارها, CTA)
  portfolio/page.tsx        صفحه نمونه‌کارها (۴ پروژه)
  contact/page.tsx          تماس با ما + فرم پیام
  shop/page.tsx             فروشگاه پلن‌ها (وردپرس / کدنویسی / سئو)
  login/page.tsx            ورود
  register/page.tsx         ثبت‌نام
  dashboard/                داشبورد کاربر (محافظت‌شده)
    page.tsx                 نمای کلی + سفارش‌ها
    profile/page.tsx         پروفایل
    settings/page.tsx        تنظیمات (زبان/تم)
    cart/page.tsx             سبد خرید + ثبت نهایی سفارش
  api/
    auth/register/route.ts   ثبت‌نام → ذخیره در data/users.json
    auth/login/route.ts      ورود → بررسی رمز و صدور کوکی نشست
    auth/logout/route.ts     خروج
    auth/me/route.ts         گرفتن کاربر جاری از روی کوکی
    orders/route.ts          ثبت و دریافت سفارش‌ها → data/orders.json
    contact/route.ts         ذخیره پیام‌های فرم تماس → data/contacts.json
components/                  کامپوننت‌های UI (Header, Footer, PlanCard, ...)
context/                     Context های زبان، تم، احراز هویت، سبد خرید
lib/data/plans.ts            اطلاعات کامل پلن‌ها (قیمت، ویژگی‌ها، توضیحات)
lib/data/projects.ts         اطلاعات ۴ پروژه نمونه‌کار
lib/data/contact-info.ts     شماره تلفن، اینستاگرام، تلگرام، واتساپ
lib/i18n/dictionaries.ts     تمام متن‌های فارسی/انگلیسی سایت
lib/db.ts                    خواندن/نوشتن فایل‌های JSON (دیتابیس ساده)
data/*.json                  «بک‌اند» فایلی: کاربران، سفارش‌ها، پیام‌های تماس
```

## نکات مهم برای شخصی‌سازی

- **اطلاعات تماس**: در `lib/data/contact-info.ts` شماره تلفن، آیدی اینستاگرام/تلگرام و لینک واتساپ را با اطلاعات واقعی جایگزین کنید.
- **پلن‌ها و قیمت‌ها**: در `lib/data/plans.ts` — هر پلن شامل عنوان، توضیح کوتاه، قیمت، مدت تحویل و لیست کامل ویژگی‌هاست.
- **نمونه‌کارها**: در `lib/data/projects.ts` — چهار پروژه (قارچ روم، AP Beauty، استایل پیکاسو، معین فایق) از قبل اضافه شده؛ در صورت داشتن اسکرین‌شات واقعی می‌توانید به‌جای پس‌زمینه رنگی در `components/ProjectCard.tsx` تصویر واقعی بگذارید.
- **متن‌ها**: تمام رشته‌های فارسی/انگلیسی سایت یک‌جا در `lib/i18n/dictionaries.ts` هستند.

## احراز هویت و ذخیره‌سازی داده

برای سادگی و اجرا بدون دیتابیس خارجی، کاربران/سفارش‌ها/پیام‌ها در فایل‌های JSON داخل پوشه `data/` ذخیره می‌شوند (`lib/db.ts`). رمز عبور با bcrypt هش می‌شود و نشست کاربر با JWT در یک کوکی httpOnly نگه داشته می‌شود.

این روش برای دمو/پروژه‌های کوچک مناسب است. برای پروداکشن و ترافیک بالا پیشنهاد می‌شود `lib/db.ts` را با یک دیتابیس واقعی (PostgreSQL، MongoDB و ...) جایگزین کنید — بقیه‌ی کد (route ها و کامپوننت‌ها) بدون تغییر زیاد کار می‌کند چون همه‌چیز از طریق همین لایه‌ی db عبور می‌کند.

سفارش‌هایی که کاربر در سبد خرید ثبت می‌کند، با دکمه «ثبت نهایی سفارش» در `data/orders.json` ذخیره می‌شوند و در داشبورد همان کاربر قابل مشاهده‌اند؛ همین فایل، مرجع شما برای بررسی سفارش‌های تازه است.

## محدودیت‌های شناخته‌شده

- صفحات داشبورد سمت کلاینت محافظت می‌شوند (ریدایرکت به ورود در صورت نبود کاربر) نه در سطح میدل‌ور — برای اکثر پروژه‌ها کافی است، ولی برای امنیت بیشتر می‌توان یک `middleware.ts` هم اضافه کرد.
- ذخیره‌سازی JSON برای همزمانی بالا مناسب نیست (رقابت در نوشتن فایل)؛ برای پروداکشن به دیتابیس واقعی مهاجرت کنید.
